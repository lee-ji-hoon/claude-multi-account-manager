# Account Bug Report

Collects diagnostic information and automatically creates a GitHub Issue.

## Instructions

### 1. Collect Diagnostic Information

Run the following commands to collect diagnostic info:

```bash
echo "=== Environment Info ==="
echo "OS: $(sw_vers -productVersion)"
echo "Python: $(python3 --version 2>&1)"
echo "Plugin: $(python3 -c "import json; print(json.load(open('${CLAUDE_PLUGIN_ROOT}/.claude-plugin/plugin.json'))['version'])" 2>/dev/null || echo 'unknown')"
echo "Accounts: $(python3 -c "import json,os; print(len(json.load(open(os.path.expanduser('~/.claude/accounts/index.json'))).get('accounts',[])))" 2>/dev/null || echo '?')"
```

```bash
echo "=== Recent Error Logs ==="
python3 "${CLAUDE_PLUGIN_ROOT}/account_manager.py" logs 2>&1 | grep -E "ERROR|WARN|FAIL" | tail -20
```

```bash
echo "=== Token Status ==="
python3 "${CLAUDE_PLUGIN_ROOT}/account_manager.py" check 2>&1
```

### 2. Confirm Symptoms with User

Use AskUserQuestion to ask the following:
- **What issue occurred?** (describe the symptoms)
- **Are there steps to reproduce?** (if any)

### 3. Create GitHub Issue

Combine the collected info and user description to create an Issue:

```bash
gh issue create --repo lee-ji-hoon/claude-multi-account-manager \
  --title "[Bug] {symptom summary}" \
  --body "$(cat <<'ISSUE_EOF'
## Symptoms
{symptoms described by user}

## Steps to Reproduce
{reproduction steps or "Not confirmed"}

## Diagnostic Information
- OS: {version}
- Plugin: v{version}
- Accounts: {N}

### Error Logs
```
{error log excerpt}
```

### Token Status
```
{token status output}
```

---
> 🤖 Auto-generated by `/account:report`
ISSUE_EOF
)" --label bug
```

### 4. Report Result

Show the Issue URL to the user.

## Notes

- `gh` CLI must be authenticated
- Do not include sensitive information (tokens, passwords) in the Issue
- accessToken, refreshToken, etc. in error logs are automatically masked
